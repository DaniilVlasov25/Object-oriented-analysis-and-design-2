## Лабораторная работа № 3
Тема: Паттерн проектирования «Наблюдатель» (Observer)

## Предметная область
Разрабатывается система событий и интерфейса персонажа (Game Dashboard) для игровой платформы. Система отслеживает состояние игрового персонажа (здоровье, мана, золото, опыт, уровень) и автоматически реагирует на изменения

## Описание проблемы
Проблема заключается в необходимости синхронизации множества компонентов интерфейса с состоянием персонажа. Если реализовать прямые вызовы методов обновления UI из класса персонажа, возникают следующие проблемы:
- Жёсткая связанность — класс GameCharacter зависит от конкретных классов интерфейса (HealthBarUI, ManaBarUI, GoldCounterUI и др.);
- Нарушение принципа открытости/закрытости — добавление нового элемента интерфейса (например, SoundManager или AchievementSystem) требует изменения кода класса GameCharacter;
- Дублирование кода — в каждом методе изменения состояния (takeDamage, gainGold, gainXP) повторяются вызовы обновления UI.

Таким образом, требуется механизм, который:
- позволяет объектам подписываться на уведомления об изменениях состояния;
- позволяет динамически добавлять и удалять подписчиков.

## Решение
Для решения проблемы в проекте реализован поведенческий паттерн проектирования Observer (Наблюдатель). Суть паттерна заключается в следующем:
- создание механизма подписки, при котором одни объекты (подписчики) следят и реагируют на события, происходящие в других объектах (издателях);
- издатель хранит список подписчиков и оповещает их об изменениях через общий интерфейс;
- подписчики могут динамически подписываться и отписываться от получения уведомлений.

В проекте созданы следующие классы:
- ISubject — интерфейс издателя, объявляющий методы attach(), detach() и notify();
- IObserver — интерфейс подписчика, объявляющий метод update();
- GameCharacter (ConcreteSubject) — класс персонажа, хранящий состояние игры и реализующий механизм подписки;
- HealthBarUI, ManaBarUI, GoldCounterUI, LevelUpNotification (ConcreteObserver) — конкретные подписчики, реагирующие на изменения состояния персонажа;
